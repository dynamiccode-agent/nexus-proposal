The Problem & Solution

Current State

  • Scheduled runs managed manually across spreadsheets, may be error-prone and time-consuming
  • No real-time visibility into seat availability, bookings, or attendance per run
  • Overbooking risk: no automated capacity enforcement per scheduled run
  • No automated confirmations, reminders, or boarding notifications
  • No-shows go unrecorded: no check-in data tied to individual runs

Our Solution

Dynamic Code will build a purpose-built transport scheduling and operations platform: a digital replacement for your spreadsheet workflow that enforces structure, automates communication, and gives you real-time operational control.

Scheduled Run → Passenger Booking → Seat Enforcement → Check-In → Live Reporting

Who Uses It

⛏️
Passengers (Miners)
Browse available scheduled runs, select a route and time, confirm their seat, check in on board via QR or NFC, and receive automated reminders.
🖥️
Admin (Office Staff)
Create and manage scheduled runs, assign capacity and drivers, monitor bookings per run in real time, track attendance, and export reports.

Why It Matters

  • Replaces the spreadsheet with a live, structured scheduling system
  • Enforces seat capacity per run with no overbooking.
  • Full real-time visibility: who's booked, who's checked in, who's a no-show
  • Automated passenger communications: zero manual follow-up
  • Designed to scale: add routes, drivers, and runs without increasing admin burden

Core Features

🗓️
Scheduled Run Management
Admin creates runs with a fixed route, date, time, capacity, and assigned driver. Passengers book seats against defined runs with no custom time selection.
📋
Passenger Booking Flow
Passenger selects route → system shows available runs → selects a run → system checks seats → confirms booking. Clean, structured, and capacity-enforced.
📊
Schedule Dashboard (Admin)
Table-based schedule view mirroring the current spreadsheet workflow, grouped by date and route. Full run management: create, edit, assign capacity, view passengers.
🪑
Live Capacity Tracking
Real-time seat availability per run. System prevents overbooking automatically. Admin can see booked vs available seats across all scheduled runs at a glance.
📱
QR + NFC Check-In
Check-in validates against a specific run and booking within a valid time window. NFC tap (premium) or QR scan (universal fallback). Offline-capable with sync on reconnect.
🔐
OTP Authentication
Passwordless login via mobile OTP through Twilio. No account setup required. First booking creates the passenger profile automatically.
💬
Automated Notifications
Email (Resend) and SMS (Twilio) for booking confirmations, run reminders, and boarding alerts. All triggered automatically with zero manual effort.
💳
Payments (Optional)
Stripe integration for pay-at-booking. Reduces no-shows, enables automated reconciliation, and syncs with Xero for accounting.

Operational Enhancements

  • No-show tracking: Unscanned bookings automatically flagged post-departure
  • Waitlist system: Passengers join a waitlist when runs reach capacity
  • Recurring run templates: Create daily/weekly routes without manual re-entry
  • CSV import: Bulk schedule setup from existing spreadsheets
  • Booking status flow: Pending → Confirmed → Completed / Cancelled / Rescheduled

Development Phases

The platform is delivered in two phases over 2 months. Each phase is a functional, production-ready release and not a prototype.

1

Core Scheduling Platform — MVP (Month 1)

Scheduled run management, admin schedule dashboard (table view by date/route), passenger booking flow, capacity enforcement, OTP login, QR check-in, no-show tracking, and automated notifications (email + SMS).

2

Payments, Automation & Reporting (Month 2)

Stripe payment integration, financial tracking, waitlist system, recurring run templates, CSV schedule import, advanced admin reporting, and optional Xero sync.

3

NFC & Enhanced Check-In (Post-Launch)

NFC tag rollout across the fleet. Passengers tap to check in. Dual system (NFC primary + QR fallback). Optional Apple/Google Wallet digital passes.

4

Mobile App (Optional Future)

Native iOS/Android app using the same backend, with native camera, NFC, push notifications, and offline-first check-in.

Project Timeline

  • Week 1–2: Discovery, data modelling, run/schedule schema design, environment setup
  • Week 3–4: Run management, admin schedule dashboard, passenger booking flow, capacity enforcement
  • Week 5–6: OTP auth, QR check-in (run-validated), notifications (email/SMS), no-show tracking
  • Week 7–8: Payments, waitlist, recurring templates, CSV import, reporting, QA, staging & handover
